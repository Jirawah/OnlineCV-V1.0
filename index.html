Concepteur / Développeur — Microservices Java & Angular

2025
Projet personnel que j'ai présenté à l'examen du TP CDA — SneakyDateReforged

Conception d’une plateforme « cheater-free » de rendez-vous entre joueurs, bâtie sur une architecture microservices Spring Boot et un frontend Angular, communicant avec les API Web Steam et Discord.
Objectif : sécuriser l’écosystème (JWT, contrôles externes) et offrir des fonctionnalités sociales (RDV, invitations, listes d’amis, profils, notifications).

Conception & modélisation :
• Recueil des besoins, user stories et backlog (vision produit / DoD).
• Wireframes puis maquettes sur Figma (design system, responsive).
• Modélisation UML : cas d’utilisation, activités, séquences, classes (contrats d’API et flux).
• Conception BDD avec la méthode Merise : MCD → MLD → MPD, normalisation et contraintes d’intégrité.
• Schéma d’architecture (microservices, sécurité JWT, passerelles d’intégration).

Réalisations principales :
• Découpage en services : auth-service, rdv-service, invitation-service, friend-service, profile-service, notification-service.
• Configuration centralisée (Spring Cloud Config), découverte de services (Eureka), communication inter-services via OpenFeign avec propagation automatique du JWT dans les en-têtes HTTP (RequestInterceptor Feign)
• Sécurisation des endpoints avec Spring Security + JWT (HMAC-256), filtres personnalisés, CORS.
• Intégration de l’API Web Steam (GetPlayerBans) à chaque login : blocage des comptes bannis (critère d’éligibilité).
• Dockerisation multi-services et orchestration via Docker Compose.
• Jeux de tests Postman et environnements dédiés pour l’API de chaque service.
• Création et intégration du bot Discord pour récupération des infos de l’utilisateur (pseudo et avatar).
• Création d’un script PowerShell pour le démarrage complet de l’app (tous les microservices) via Docker Compose.

Frontend Angular (dockerisé en microservice) :
• Mise en place d’un frontend Angular 19 en SPA avec composants standalone, routing structuré par layouts (public-layout / app-layout).
• Formulaires réactifs (Reactive Forms) pour l’inscription, la connexion et la réinitialisation de mot de passe, avec validations avancées (Steam ID, Discord connecté, champs requis).
• Gestion du JWT côté client (sessionStorage) avec interceptors HTTP (authentification et gestion d’erreurs) et guards de navigation (pages publiques / pages authentifiées).
• Composants réutilisables pour le header, les listes d’amis (aside right), les cards de profils (aside left) et les bandeaux d’alerte (erreur/succès).
• Intégration du flux d’onboarding complet Steam + Discord dans le formulaire d’inscription (affichage des états, pseudo Discord détecté, blocage du bouton tant que les prérequis ne sont pas remplis).
• Styles SCSS organisés (BEM-like) et responsive pour les pages authentifiées et non authentifiées.
